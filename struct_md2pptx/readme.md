# Markdown 转 PowerPoint 脚本使用说明

本脚本可将特定格式的 Markdown 报告自动转换为 PowerPoint 演示文稿，支持使用自定义模板（主题），并会忽略文档结尾的“PPT结束”标记。

## 功能特点

- 按 `---` 分隔符将 Markdown 拆分为多个页面，每个页面生成一张幻灯片。
- 自动识别封面页（第一个包含 `**标题：**`、`**副标题：**` 等字段的页面）并使用模板的标题幻灯片布局。
- 支持字段：
  - `**标题：**`（封面页大标题）
  - `**副标题：**`（封面页副标题）
  - `**汇报人/机构：**`（封面页作者/机构信息）
  - `**内容：**`（幻灯片正文，支持无序列表 `- 项目`）
  - `**备注/解说词：**`（演讲者备注，添加到 PPT 备注栏）
- 正文中的 `**粗体**` 会被保留为加粗格式。
- 支持传入 `.pptx` 模板文件，新生成的幻灯片将继承模板的母版、主题、颜色、字体和背景。
- 自动忽略以 `---` 分隔且内容仅为 `**（PPT结束）**` 的页面块。

## 依赖安装

本脚本需要 `python-pptx` 库。

```bash
pip install python-pptx
```

## 使用方法

### 基本用法（无模板，使用默认空白样式）

```bash
python struct_md2pptx.py 1.md output.pptx
```

### 使用主题模板

```bash
python struct_md2pptx.py 1.md output.pptx --template theme.pptx
```

其中 `theme.pptx` 是您预先设计好主题（字体、颜色、背景等）的 PowerPoint 模板文件。

## Markdown 格式规范

您的 Markdown 文件应遵循以下结构（示例来自提供的 `1.md`）：

```markdown
---

### PPT 1：封面

**标题：** 标题1
**副标题：** 副标题1
**汇报人/机构：** [您的名字/机构]

**备注/解说词：**
各位好。今天要和大家探讨...

---

### PPT 2：标题2

**内容：**
- 内容1111...
- 内容222。
- 现实：3333...

**备注/解说词：**
让我们先走进一个极具冲击力的场景...

---

**（PPT结束）**
```

### 格式要点

1. **页面分隔**：每个幻灯片的内容块之间用单独一行的 `---` 隔开。
2. **页面标题**：每个块的第一行应为 `### PPT N：标题内容`，其中“标题内容”将作为该幻灯片的标题。
3. **字段标记**：字段名称必须使用 `**字段名：**` 的格式，中英文冒号均可，但建议保持一致（示例中为中文冒号 `：`）。
4. **内容列表**：在 `**内容：**` 字段下方，以 `- 项目文本` 形式书写，每行一个项目，支持内联 `**粗体**`。
5. **备注**：`**备注/解说词：**` 字段的内容会填入幻灯片的备注窗格。
6. **结束标记**：文档最后可以放置一行 `---` 然后单独一行 `**（PPT结束）**`，脚本会自动忽略该页，不会生成空白幻灯片。

## 示例命令

假设您有一个名为 `report.md` 的 Markdown 文件，一个名为 `company_theme.pptx` 的模板文件，执行：

```bash
python struct_md2pptx.py report.md presentation.pptx --template company_theme.pptx
```

将生成 `presentation.pptx`，其中每个幻灯片沿用 `company_theme.pptx` 的视觉主题，内容完全来自 `report.md`。

## 注意事项

- 脚本默认使用模板中的 **第一个布局**（索引 0，通常为标题幻灯片）作为封面，**第二个布局**（索引 1，通常为标题+内容）作为普通内容页。如果模板中没有这些布局，脚本会自动回退到空白布局并手动添加文本框。
- 模板文件应确保至少包含两个布局，或至少存在标题占位符和内容占位符。
- 如果省略 `--template` 参数，脚本将使用 `python-pptx` 的内置默认空白模板。
- 所有字段名称和冒号必须与脚本中的正则匹配完全一致（包含中文冒号 `：`）。
- 图片、表格等复杂元素暂不支持，仅支持文本和列表。

## 常见问题

**Q：提示 `unrecognized arguments: --template`？**  
A：请确保您保存了最新版本的脚本（支持 `-t/--template` 参数），并且命令行参数格式正确（两个减号）。

**Q：生成的 PPT 中标题或内容位置错乱？**  
A：这通常是因为模板中的占位符索引与脚本预期不一致。您可以修改脚本中 `create_cover_slide` 和 `add_content_slide` 函数，调整占位符的获取方式（例如通过 `placeholder_format.idx` 或 `shape.name` 来定位）。

**Q：如何让生成的幻灯片使用模板中的特定母版？**  
A：脚本默认使用布局索引 0 和 1。如果您的模板中有多个母版，可以在脚本中指定其他布局索引，例如 `prs.slide_layouts[2]`。
